### 开发tip:

+ 不要相信任何开发：一定要保留老版本，一旦新版本出问题，确保随时可以回滚

+ 不要相信任何测试：不论线下怎么测试，上线以后还是有可能出问题。因此需要做灰度发布，新版本先开放给一小部分用户进行真实测试，没问题再向所有用户开放。

+ 不要相信任何应用：任何应用都可能崩溃，必要的时候做服务降级和限流，以保证核心业务可用。

+ 不要相信任何硬件：业务做跨机房部署，进行容灾。
